數據庫測驗解答 - 第 4 題

背景資料：電影中心使用數據庫表 TINFO 來儲存戲票資料。

欄名描述例子
MID電影的識別碼M001
MNAME電影名稱狂舞派
MTIME日期和時間14/02/2023 14:00
CID戲院的識別碼S2
CTYPE戲院類別A
TN戲院座位的數目150
SNO座位編號18

(i) 運用集合函數完成以下 SQL 語句，以列出每場電影已售出貴賓戲票的數量。

提示：座位編號為 1 至 10 為貴賓戲票。

SELECT CID, MTIME, ____________________
FROM TINFO
WHERE SNO <= 10
GROUP BY CID, MTIME
解答：
應填入 COUNT(SNO) 或 COUNT(*)。
解釋： 題目要求計算「數量」，因此需要使用 COUNT 聚合函數來計算符合條件（SNO <= 10）的記錄數目。

(ii) 寫出在 TINFO 的 CID 和 MTIME 上建構索引「SHOWTIME」的 SQL 語句。

解答：
CREATE INDEX SHOWTIME ON TINFO (CID, MTIME);
解釋： 使用 CREATE INDEX [索引名稱] ON [資料表] ([欄位1], [欄位2]...); 的語法來建立複合索引。

(iii) 舉出使用索引「SHOWTIME」的一個好處。

解答：
1. 加快查詢速度： 當執行如 (i) 部份中需要根據 CID 和 MTIME 進行搜尋 (WHERE) 或分組 (GROUP BY) 的查詢時，數據庫可以直接透過索引快速定位數據，而不需要掃描整張資料表 (Full Table Scan)，從而大幅提升檢索效率。
