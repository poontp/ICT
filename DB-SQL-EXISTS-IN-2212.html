SQL 查詢原理：EXISTS vs IN

EXISTS 的核心邏輯：外部查詢每讀取一筆資料，就去執行一次子查詢。只要在子查詢找到第一筆符合的資料，就會立刻停止搜尋（Early Exit）並回傳 TRUE。

SELECT CustomerName FROM Customers
WHERE EXISTS (
    SELECT 1 FROM Orders
    WHERE Orders.CustomerID = Customers.CustomerID
);

IN 的核心邏輯：先完整執行一次子查詢，將結果收集成一個「暫存清單 (IN List)」，然後外部查詢再去比對這個清單。

SELECT CustomerName FROM Customers
WHERE CustomerID IN (
    SELECT CustomerID FROM Orders
);

EXISTS 適用情境

  • 子查詢資料量非常大時（例如 Orders 表有百萬筆）。
  • 因為 EXISTS 只要找到「第一筆」符合的就會提早結束 (Early Exit)，不用掃描整個子查詢表。
  • 外部查詢資料量小，子查詢資料量大時，效能極佳。
  • 可以利用子查詢表上的索引（Index）快速定位。
白話文：「老闆(外部查詢)問員工(子查詢)：『這客戶有買過東西嗎？』員工翻紀錄，一看到有買過就馬上回報『有！』，不用管他到底買了幾次。」

IN 適用情境

  • 子查詢資料量很小時（例如查出特定的幾個 VIP 客戶 ID）。
  • 因為 IN 會先把子查詢「全部執行完」並放在記憶體中變成一個清單。
  • 外部查詢資料量大，子查詢資料量小時，效能較好。
  • 如果清單是固定的（例如 IN (1, 2, 3)），資料庫最佳化器處理起來非常快。
白話文：「老闆請員工先把『所有買過東西的客戶名單』整理成一張表(IN List)，然後老闆自己拿著客戶名冊，一個一個對照那張表。」
