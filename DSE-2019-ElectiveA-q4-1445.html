數據庫測驗解答 - 第一題 (Q1)

資料表結構參考

BROUTE: BID (巴士路線), BNO (巴士數目)
EMPLOYEE: EID (員工識別碼), ENAME (姓名), SALARY (薪酬), DSER (首天上班日期)
DRIVER: EID (員工識別碼), BID (可駕駛路線)
(a) 列出可駕駛巴士路線「1A」的員工的姓名和識別碼，並按他們的薪酬降序排列。
SELECT E.ENAME, E.EID FROM EMPLOYEE E, DRIVER D WHERE E.EID = D.EID AND D.BID = '1A' ORDER BY E.SALARY DESC;
解釋：
  • 需要從 EMPLOYEE 表獲取姓名 (ENAME) 和識別碼 (EID)。
  • 需要連接 DRIVER 表來過濾可駕駛路線 (BID) 為 '1A' 的記錄。
  • 使用 ORDER BY E.SALARY DESC 實現薪酬降序排列。
(b) 列出於 2009 年加入公司的員工的最高薪酬。
SELECT MAX(SALARY) FROM EMPLOYEE WHERE YEAR(DSER) = 2009; -- 或者使用日期範圍 -- -- WHERE DSER >= '1/1/2009' AND DSER <= '31/12/2009' --
解釋：
  • 使用聚合函數 MAX(SALARY) 找出最高薪酬。
  • 使用 YEAR(DSER) = 2009 或日期範圍條件來篩選出 2009 年入職的員工。
(c) 列出不能駕駛任何巴士路線的員工的姓名和識別碼。
SELECT ENAME, EID FROM EMPLOYEE WHERE EID NOT IN (SELECT EID FROM DRIVER);
解釋：
  • 這是一個尋找「不存在於」關聯表中的記錄的問題。
  • 子查詢 SELECT EID FROM DRIVER 找出所有能駕駛巴士的司機的 EID。
  • 主查詢使用 NOT IN 找出不在上述列表中的員工（即非司機）。
(d) 某些巴士路線的司機數目少於巴士數目。列出這些巴士路線。
CREATE VIEW DRIVERNO AS SELECT BROUTE.BID, COUNT(*) AS TOTAL FROM BROUTE, DRIVER WHERE BROUTE.BID = DRIVER.BID GROUP BY BROUTE.BID; SELECT BROUTE.BID FROM BROUTE, DRIVERNO WHERE BROUTE.BID = DRIVERNO.BID AND TOTAL < BNO;
解釋：
  • 第一步： 建立一個名為 DRIVERNO 的檢視表 (View)。它連接了 BROUTE 和 DRIVER 表，並按路線 (BID) 分組，計算出每條路線的總司機人數，並將其命名為 TOTAL。
  • 第二步： 從 BROUTE 表和剛建立的 DRIVERNO 檢視表中查詢。透過 BROUTE.BID = DRIVERNO.BID 將它們連接起來，並加上條件 TOTAL < BNO，從而篩選出司機總數 (TOTAL) 少於該路線巴士數目 (BNO) 的巴士路線。
(e) 參考下列檢視：
(i) V1 的目的是什麼？
(ii) 寫出一個 SQL 指令以求 (AVG1 - AVG2)。
(i) V1 的目的：

計算在 2005 年 1 月 1 日之前入職的司機的平均薪酬。


(ii) SQL 指令：
SELECT V1.AVG1 - V2.AVG2 FROM V1, V2;
解釋：
  • V1 視圖計算了 2005 年前入職司機的平均薪酬 (AVG1)。
  • V2 視圖計算了 2005 年或之後入職司機的平均薪酬 (AVG2)。
  • 直接從這兩個視圖中選取並相減即可得出結果。由於視圖只返回單一值，可以進行簡單的交叉連接。
