數據庫測驗解答 - 第二題 (Q2)

資料表結構參考

MEMBER: MID (會員識別碼), MNAME (英文姓名), MTYPE (會員類別: G/O), CREDIT (積分點), JDATE (入會日期)
VIDEO: VID (視頻識別碼), VTITLE (視頻英文片名), RFEE (視頻租賃費)
RENTAL: MID (會員識別碼), VID (視頻識別碼), RDATE (租賃日期)
(a) RENTAL 的主關鍵碼是什麼？
解答：

MID + VID + RDATE （即由會員識別碼、視頻識別碼和租賃日期組成的複合鍵）

解釋：

因為同一個會員可能會在不同的日期租借同一個視頻，所以單獨的 MID 或 VID 不足以唯一識別一筆租賃記錄。必須結合會員、視頻和具體的租借日期才能確保記錄的唯一性。

(b) 列出積分少於 100 的普通會員的英文姓名和會員識別碼。
SELECT MNAME, MID FROM MEMBER WHERE MTYPE = 'O' AND CREDIT < 100;
解釋：
  • 從 MEMBER 表中選取 MNAME 和 MID。
  • 條件 1：普通會員，即 MTYPE = 'O'。
  • 條件 2：積分少於 100，即 CREDIT < 100。
(c) 找出視頻「Nice Face」的總租賃費。
SELECT SUM(V.RFEE) FROM VIDEO V, RENTAL R WHERE V.VID = R.VID AND V.VTITLE = 'Nice Face';
解釋：
  • 需要連接 VIDEO 表（獲取租金和片名）和 RENTAL 表（獲取租借次數/記錄）。
  • 使用 SUM(V.RFEE) 將所有相關的租賃費用加總。
  • 條件為視頻名稱 V.VTITLE = 'Nice Face'。
(d) 列出從未被租賃的視頻的英文片名。
SELECT VTITLE FROM VIDEO WHERE VID NOT IN (SELECT VID FROM RENTAL);
解釋：
  • 子查詢 SELECT VID FROM RENTAL 找出所有曾經被租借過的視頻 ID。
  • 主查詢使用 NOT IN 找出不在上述列表中的視頻，即從未被租借過的視頻，並顯示其 VTITLE。
(e) 公司為慶祝週年紀念，送贈額外 50 個積分點予 2017 年前入會的會員。
UPDATE MEMBER SET CREDIT = CREDIT + 50 WHERE YEAR(JDATE) < 2017; -- 或者使用日期格式 -- -- WHERE JDATE < '1/1/2017'; --
解釋：
  • 這是一個數據更新操作，使用 UPDATE 語句。
  • SET CREDIT = CREDIT + 50 表示在原有積分上增加 50。
  • 條件為入會年份早於 2017 年。
(f) 以下 SQL 語句的目的是什麼？
SELECT MID FROM MEMBER WHERE MTYPE = "G" AND JDATE < ALL (SELECT JDATE FROM MEMBER WHERE MTYPE = "O")
解答：

列出所有入會日期早於所有普通會員（最早期入會的普通會員）的黃金會員的會員識別碼 (MID)。

解釋：
  • 子查詢找出所有普通會員 (MTYPE = "O") 的入會日期。
  • < ALL 表示必須小於子查詢結果中的每一個值，也就是說，必須比最早入會的普通會員還要早入會。
  • 主查詢則篩選出符合此條件的黃金會員 (MTYPE = "G")。
